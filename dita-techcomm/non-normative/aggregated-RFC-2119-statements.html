Aggregated RFC-2119 statements

This appendix contains all the normative statements from the DITA for Technical Content 2.0 specification. They are aggregated here for convenience in this non-normative appendix.

Aggregated error statements

DITAERR-001It is an error if the value of this attribute is not an unsigned integer.
DITAERR-002In this case, the implementation might give an error message and might recover by ignoring this attribute.
DITAERR-003It is an error to use parse="xml" anywhere other than within <foreign> or a specialization thereof.
DITAERR-004It is an error to use parse="xml" anywhere other than within <foreign> or a specialization thereof.

Aggregated RFC-2119 statements

DITAREQ-001If the referenced topic is not a <glossentry> topic (or a specialization of <glossentry>), the title of the topic SHOULD be displayed.
DITAREQ-002Otherwise, if <abbreviated-form> is located in an introductory context, and the referenced topic contains a non-empty <glossSurfaceForm> element, processors SHOULD render the contents of the <glossSurfaceForm> element from the referenced <glossentry> topic.
DITAREQ-003Otherwise, if <abbreviated-form> is located in an introductory context, processors SHOULD render the contents of the <glossterm> element from the referenced <glossentry> topic.
DITAREQ-004Otherwise (in non-introductory contexts), if the referenced <glossentry> topic contains a non-empty <glossAcronym> element, processors SHOULD render the abbreviated form of the term by displaying the contents of the <glossAcronym> element from the referenced <glossentry> topic.
DITAREQ-005Otherwise, processors SHOULD render the contents of the <glossterm> element from the referenced <glossentry> topic.
DITAREQ-006Processors which render the content of <stepsection> elements among the <step> elements MUST NOT number the <stepsection> elements.
DITAREQ-007When the <equation-number> element has empty or whitespace-only content, the equation number SHOULD be generated.
DITAREQ-008When the <equation-number> element has non-whitespace-only content the content SHOULD be used as the equation number.
DITAREQ-009For explicit numbers the content of the element SHOULD be the number value without any surrounding punctuation, for example, "3.2a" rather than "(3.2a)".
DITAREQ-010Processors SHOULD add punctuation or decoration to the number as appropriate.
DITAREQ-011Processors SHOULD process the MathML as though the <m:math> element had occurred directly in the content of the containing <mathml> element.
DITAREQ-012Processors SHOULD preserve line the breaks and spaces that are present in the content of a <codeblock> element.
DITAREQ-013Processors SHOULD preserve the line breaks and spaces that are present in the content of a <msgblock> element.
DITAREQ-014Processors SHOULD process the SVG as though the <svg> element had occurred directly in the content of the containing <svg-container> element.
DITAREQ-015Processors SHOULD preserve the line breaks and spaces that are present in the content of a <screen> element.
