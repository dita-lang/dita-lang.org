Release management domain

The release-management domain elements contain human-authored information about the changes that have been made to a DITA topic or map. A processor can retrieve this information and use it to assemble documents or topics that contain release note information.

  • change-completed
    The <change-completed> element indicates the date on which a change was completed.
  • change-historylist
    The <change-historylist> element contains individual release notes, as represented by the <change-item> element.
  • change-item
    The <change-item> element specifies the record of a single change to a DITA topic or map.
  • change-organization
    The <change-organization> element specifies the name of the organization that required a change.
  • change-person
    The <change-person> element to specifies the name of the person who made a change.
  • change-request-id
    The <change-request-id> element to specifies an identifier associated with the change request, such as an issue ID or ticket number.
  • change-request-reference
    The <change-request-reference> element provides information that links the change to an external tracking system.
  • change-request-system
    The <change-request-system> element specifies the name of an information system that manages or serves the referenced change request, for example, an issue tracking system.
  • change-revisionid
    The <change-revisionid> element specifies a revision ID string that can identify the change.
  • change-started
    The <change-started> element specifies the date on which the change was initiated.
  • change-summary
    The <change-summary> element summarizes the details of a change.
