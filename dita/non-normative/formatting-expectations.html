Formatting expectations

DITA is a standard that supports the creation of human-readable content. Accordingly, DITA defines fundamental document components. Since there is a reasonable expectation that such document components be rendered consistently, we suggest the following formatting conventions.

Table 1. Formatting expectations for DITA elements
Element Suggested formatting
<dd> See <dl>.
<dl> Apply the following conventions:
  • The term (<dt>) is against the starting margin of the page or column.
  • The description or definition (<dd>) is either indented and on the next line or on the same line after the term.
  • The <dlhead> looks like a table heading row.
<dlhead> See <dl>.
<dt> See <dl>.
<li> Apply the following conventions:
  • In ordered lists, list items are indicated by numbers or alphabetical characters.
  • In unordered lists, list items are indicated by bullets or dashes.
<ol> See <li>.
<sl> See <sli>.
<sli> Apply the following conventions:
  • The content of each simple list item is placed on a separate line.
  • The lines are not distinguished by numbers, bullets, or other icons.
