Specialization elements

Several DITA elements exist either for architectural reasons or for support of specialized markup yet to be designed.

There is little need to use these elements unless your content can take advantage of the semantic distinction. For example, a discussion of signals on a gate of an integrated logic circuit might use the <state> element to represent either on or off conditions of that gate.

  • data
    Data is a generic component that represents metadata within a topic or map. Complex metadata is represented by nested data structures.
  • foreign
    Foreign content is non-DITA content, such as MathML, SVG, or Rich Text Format (RTF).
  • no-topic-nesting
    The <no-topic-nesting> element is a placeholder in the DITA architecture.
  • state
    The <state> element represents a named property that has a variable state value.
  • unknown
    The <unknown> element is an open extension that enables information architects to incorporate XML fragments in DITA topics.
