Changes from previous versions

The following topics outline the changes from earlier versions of DITA to the current version. Each version of the DITA specification is designed to be backwards-compatible with applications that conform to earlier versions of the DITA specification.

  • Changes from DITA 1.2 to DITA 1.3
    DITA 1.3 adds a number of new features and refinements to DITA.
  • Changes from DITA 1.1 to DITA 1.2
    DITA 1.2 added a number of new features to DITA, including indirect addressing using map-defined keys; the ability to define content-model constraints for DITA document types; specializations for learning content and the machine industry; and taxonomies, ontologies, and controlled vocabularies. Other refinements include extended markup for glossaries and terminology.
  • Changes from DITA 1.0 to DITA 1.1
    The DITA 1.1 specification was designed to be backwards-compatible with applications that conform to the DITA 1.0 specification.
